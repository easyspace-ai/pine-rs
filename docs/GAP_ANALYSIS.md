# Pine Script v6 对齐差距清单（pine-rs）

本文档整理 **pine-rs** 与 **TradingView Pine Script v6**（以仓库内 [`docs/pinescriptv6`](./pinescriptv6) 官方文档镜像为口径）之间**尚未对齐**的项，并粗估若追求**全面对齐**的整体难度。

- **执行层 checklist（按条目标记进度）**请以 **[`V6_ALIGNMENT.md`](./V6_ALIGNMENT.md)** 为准；本文件保留 **背景叙述与难度分析**。
- 产品与阶段目标以 [`AGENTS.md`](../AGENTS.md) 为准；此处**不**等同于当前 Phase 的必做列表。
- 「全对齐」指：**语法可接受典型 TV 脚本、静态规则与运行时行为与文档一致、可验证**；不承诺未公开的 TV 实现细节。

---

## 1. 语法 / 解析层

| # | 主题 | 官方 v6（文档与常见脚本） | 当前 pine-rs 大致情况 |
|---|------|---------------------------|------------------------|
| 1 | `switch` | 表达式 + 缩进内 **`pattern => expr`**；可无被匹配表达式；默认分支常为单独 **`=>`** | **主形态已打通**；parser 与 eval 已支持 scrutinee / guard 两种形式；剩余工作是补更厚的脚本样本 |
| 2 | 用户函数 | **`name(params) =>`、export/method 与 `=>` 一体** | **已支持官方 `name(params) =>` 主路径**；`fn` 仍保留兼容层；闭包与更深 TV 语义仍未完全对齐 |
| 3 | `import` 路径 | 如 **`user/lib/1 as m`**（路径段，不必是字符串） | **Parser 已支持路径段语法**；eval 已支持真实文件加载；仍需继续补更完整官方模组样本 |
| 4 | `export` | **`export` 修饰整段声明**（含参数表与 `=>`） | **已支持 `export foo(...) => ...` 与 `export name = expr`**；仍需继续加脚本级回归而不是只靠单测 |
| 5 | `enum` | 独立 **`enum`** 与字段 | **无** keyword / stmt |
| 6 | `for ... in` | **`for x in arr`、`for [i,v] in arr`**、矩阵行等 | **已支持 array / series 与元组解构**；矩阵相关形态仍未闭环 |
| 7 | `else if` vs `elif` | 手册示例常见 **`else if`** | **`else` + `if` 已正确处理**；`elif` 只是兼容性词法残留 |
| 8 | 复合赋值 | **`%=`** 等 | `AssignOp` 已含主要复合赋值；更广泛 TV 兼容性仍需持续核对 |
| 9 | 类型限定符 | **`const` / `simple` / `series`** 等声明前缀 | **无**多词类型头；部分仅为 **`Ident`** |
| 10 | 绘图等类型名 | **`label`、`line`、`box`、`table` 等作类型** | 可能落 **`User` 名**；与官方规则**未保证一致** |
| 11 | 编译器注解 | `//@version=6`、`//@description` 等 | **`//` 丢弃**，**无 AST**（工具链缺口） |
| 12 | 脚本入口 | `indicator()`、`strategy()`、`library()` | 多为 **普通调用表达式**；**个别边界需脚本测** |

## 2. 执行 / 语义 / 运行时

| # | 主题 | 期望 | 当前 pine-rs 大致情况 |
|---|------|------|------------------------|
| 1 | **用户函数 UDF** | 定义、调用、（按需）闭包与调用点隔离 | **官方 `=>` 形态已接入主路径**；调用点隔离已有回归；剩余缺口在更深闭包与 TV 全语义 |
| 2 | `method` / UDT | 与 TV 对象与方法解析一致 | **`TypeDef` / `MethodDef` 在执行侧多跳过或依赖未接好的语义阶段** |
| 3 | **`pine-sema`** | 类型、作用域、`var`/`varip`、导出规则等与执行一致 | Crate 存在；**eval 主线未由 analyze 驱动** |
| 4 | `switch` 语义 | 与官方 **表达式 `switch`**、**`=>` 默认支**一致 | **主语义已打通**；剩余工作是增加更复杂脚本样本与边界锁定 |
| 5 | 逐 bar runner | 稳定错误与可复现输出 | Runner **按 bar 设置 `bar_index`**；系列 **`push_to_series` 按 bar 合并**，利于 `var` 历史下标；**错误与输出形态仍在收紧** |
| 6 | **`pine-vm`** | 可选高性能后端 | **已完成 44 组黄金 + 44 组 script parity**；CLI / API 已提供切换与回退基础，剩余任务转为默认路径切换前的覆盖加厚 |
| 7 | `request.*` / 多周期 | 文档与真实脚本大量依赖 | [`AGENTS.md`](../AGENTS.md)：`request.security` **Phase 5 前可 `na` 占位**；**与 TV 不等价** |

## 4. 若「全对齐」，难度大吗？

结论：**工作量很大，且呈「前陡后缓、尾部无限」**——语法与 UDF 是**可规划的大块**；与 TV **数值与行为逐函数一致**、**全标准库 + request 栈**是**长期工程**。下面按层次说明。

### 4.1 解析器对齐（难度：**中高**，相对可边界化）

- **痛点**：`switch`（`=>` arm、无表达式形式）、**`=>` 函数族**（与 `fn` 统一到一类 AST）、**`import` 路径**、**`export` 声明形态**、**`for...in`**、**`enum`**。
- **性质**：大多是**文法与 AST 重构 + 大量 snapshot/回归测试**；不依赖行情数据源；**难度可控但需要集中时间**。
- **风险**：缩进块与 `=>` 混排易出**歧义与错误恢复**问题，调试成本高。

### 4.2 语义分析 + 运行时规则（难度：**高**）

- Pine 的 **`series` / `simple` / `const`**、**`var`/`varip`**、**函数调用点上的 history 隔离**、**分支对齐**，与「普通脚本语言」差别大。
- **性质**：需要 **`pine-sema` 真接入 eval**（或生成等价 IR），否则 Parser 对齐后仍会出现**静默错语义**。
- **难度**：属于**核心语言设计工作**；估时往往以**季度**计，而非几天。

### 4.3 用户自定义函数与 `method`（难度：**高**，且与 4.2 强耦合）

- TV 脚本大量使用小函数；**无 UDF 则「语法对齐」意义打折**。
- 需：**闭包或等价机制**、**参数绑定**、**返回与 `na`**、**与 sema 的 series 规则一致**；可选 **递归限制**、**栈深**。
- **性质**：实现量低于「整个 stdlib」，但**语义 bug 代价大**，必须靠**系统性测试**托底。

### 4.4 标准库与内置 API（难度：**中高 → 极高**）

- **`ta.*` / `math.*` / `str.*`**：单函数可逐个做；**全量对齐**依赖黄金测试与参考实现，**工时线性增长**。
- **`request.security`、策略撮合、`strategy.*` 全语义**：超出「表达式求值」，涉及**数据馈送与市场模型**；[`AGENTS.md`](../AGENTS.md) 已将其与 Phase 路线挂钩，**不宜低估**。

### 4.5 字节码 VM（难度：**中高（工程）**，当前已进入切换前阶段）

- VM **不自动解决**语义问题；价值在**性能、稳定执行与后续主路径收敛**。
- 当前仓库里，VM 已经不再只是“未来优化项”：同批黄金与 script parity 已对齐。
- 后续关键问题不再是“能不能跑”，而是：
  - 默认路径切到 VM 前，是否保留 **eval 回退**
  - 新增关键脚本是否默认进入 **VM 回归**
  - CLI / API 已有明确切换开关后，是否继续补厚入口层回归

### 4.6 总体判断（务实结论）

| 目标 | 难度感知 | 说明 |
|------|-----------|------|
| **语法上「能解析多数公开 v6 脚本结构」** | 中高 | Parser + 测试密集；**6～12 周量级**（随人力与测试基线浮动） |
| **语义上「自洽 + 黄金测试覆盖核心指标/ta」** | 高 | sema + UDF + runner；**按月～按季** |
| **与 TV 行为「全库、全边界」一致** | 极高 | 开放边界多；宜以**文档 + 抽样 + 黄金**定义「完成」 |
